HTTP Status Codes Reference

In-depth guides for the most common HTTP status codes. Each page explains what the code means, common causes, practical troubleshooting steps, code examples, and related tools.

2xx

Success

Success responses indicate that the client's request was received, understood, and accepted.

200
OK
The standard success response. The request has succeeded and the server returned the requested resource or result.
Read full guide →
201
Created
A new resource was successfully created. Typically returned after POST requests with a Location header pointing to the new resource.
Read full guide →
3xx

Redirection

Redirection responses tell the client it needs to take additional action to complete the request, usually following a URL in the Location header.

301
Moved Permanently
The URL has permanently changed. Search engines transfer link equity to the new URL. Essential for domain migrations and HTTPS upgrades.
Read full guide →
302
Found
Temporary redirect. The original URL remains canonical. Used for A/B tests, login flows, and geo-based routing.
Read full guide →
304
Not Modified
The cached version is still valid. Saves bandwidth by telling the client to reuse its cached copy via ETag or Last-Modified validation.
Read full guide →
4xx

Client Errors

Client error responses indicate that the request contains bad syntax, invalid credentials, or otherwise cannot be fulfilled by the server.

400
Bad Request
The server cannot process the request due to malformed syntax. Common with invalid JSON, missing parameters, or corrupt cookies.
Read full guide →
401
Unauthorized
Authentication is required. The request lacks valid credentials — token missing, expired, or invalid. Not to be confused with 403.
Read full guide →
403
Forbidden
Access denied. The server knows who you are but you do not have permission. Often caused by file permissions, WAF rules, or role restrictions.
Read full guide →
404
Not Found
The most recognized HTTP error. The URL does not match any resource on the server. Often fixable with 301 redirects for moved pages.
Read full guide →
405
Method Not Allowed
Wrong HTTP method for this resource. You sent POST when only GET is supported, or DELETE when the resource is read-only.
Read full guide →
429
Too Many Requests
Rate limited. You sent too many requests too quickly. Handle with exponential backoff and respect the Retry-After header.
Read full guide →
5xx

Server Errors

Server error responses indicate that the server failed to fulfill a valid request. The problem is on the server side, not the client.

500
Internal Server Error
Generic server failure. Something went wrong but the server cannot be specific. Debug via error logs — the answer is always there.
Read full guide →
502
Bad Gateway
The proxy/gateway received an invalid response from the upstream server. Usually means the backend crashed or is not running.
Read full guide →
503
Service Unavailable
The server is temporarily unable to handle requests. Used during maintenance, overload, or deployment. Always temporary.
Read full guide →
504
Gateway Timeout
The upstream server did not respond in time. Caused by slow queries, long-running operations, or network issues between servers.
Read full guide →
